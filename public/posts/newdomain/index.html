Update#

The Images not loading issue was because of hugo baseUrl configuration. This varialbe if set to domain or custom link it resolves the links relative to that url. So if I leave it empty basUrl = "" and ran with hugo server command it automatically assumes “localhost:1313” for base url. but if set to another url like example.com it try it use it. but my static site wasnt deployed using hugo framework from cloudflare I just build hugo site on windows and push it to github repo and cloudflare pages will statically serve the pages in “/public” folder.

So upon changin baseUrl to “/” just works and fixed the issue.

So ig now I have one less issue to fix.

Another Update as on 08/12/2024#

It turns out these blacklist will automatically clear my domain after 5 days Fortunately it seems to fixed itself and now I dont have to wait for 5 days and I can load my website without vpn now.

Notes for self#

When building site with hugo always use with these flags if you dont want to regret later.

hugo server --baseURL https://example.org/ --appendPort=false

additionally you can also add baseUrl="https://example.org" in your hugo.yaml
